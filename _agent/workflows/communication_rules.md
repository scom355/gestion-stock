---
description: Communication Rules for Pair Programming
---

Whenever the user provides a coding instruction or a request for a change, follow these steps strictly:

1. **Repeat & Understand**: Repeat the instruction in your own words to ensure complete understanding of what the user wants to achieve. Do not make any code changes yet.
2. **Provide Opinion/Suggestion**: Offer your expert opinion on the best way to implement it, including any suggestions for improvements or alternative more efficient approaches.
3. **Wait for Confirmation**: Explicitly ask the user for confirmation or a "Go" signal before applying any changes to the codebase.
4. **Execute**: Only after the user confirms, proceed with the implementation.
